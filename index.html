CS 61C

计算机体系结构与机器级程序

适合已经有编程基础、想理解程序如何在硬件上执行的学习者。课程强调动手项目，从 C 与 RISC-V 逐步过渡到 CPU、缓存和并行计算。

  • 核心主题：C、RISC-V、调用约定、流水线、Cache、虚拟内存、SIMD/OpenMP。
  • 学习产出：能读写汇编、理解性能瓶颈，并完成体系结构相关项目。
  • 建议前置：至少完成一门编程入门课和基础数据结构训练。

EECS 70

离散数学与概率论

适合补齐 CS 理论基础的学习者。它连接算法、机器学习、密码学、网络和系统设计中常见的数学语言。

  • 核心主题：证明方法、归纳、图、模运算、计数、条件概率、随机变量。
  • 学习产出：能阅读更形式化的算法和概率分析材料。
  • 建议前置：熟悉基础编程，愿意投入时间练习证明与推导。

如何使用这个归档

不要只收藏链接。像完成真实课程一样推进：看材料、做实验、交付项目、复盘错题。

1. 先确定课程定位 2. 按周推进材料 3. 优先完成项目 4. 用笔记沉淀结果

按课程而不是按资料学习

一门好课程已经替你组织好了概念顺序、练习密度和项目目标。尽量跟随课程节奏，而不是在不同资源之间频繁切换。

把项目当作主线

CS 课程的关键收获往往来自实验和项目。讲义可以帮助理解，但真正的掌握通常发生在调试和实现过程中。

保留英文术语

中文整理用于降低阅读门槛，关键术语仍建议同时记住英文，方便查资料、读论文和参与英文社区讨论。

持续校对与修复

归档页会逐步修复链接、翻译、脚本和排版问题。如果页面行为异常，优先检查缓存和 GitHub Pages 部署状态。

归档内容

面向长期访问整理课程资料，尽量减少外部脚本和易失链接对学习过程的影响。
