练习5：Git乐趣

在本练习中，您将获得实验室的Git仓库（"repo"）并使用各种Git命令。

本练习的说明假设您使用Vim编辑文本文件。有关vim的快速入门指南，请参阅附录中的Vim基础部分。

如果您想使用其他文本编辑器，请直接使用。但是，工作人员将无法为您提供vim以外的编辑器的支持。一些示例：

  • Nano: simple and beginner-friendly compared to Vim. It provides a helpful list of commands at the bottom of the interface (the ^ means the Ctrl key). Ships with many UNIX distributions (e.g. macOS, Ubuntu Linux). Open with nano file.txt.
  • Visual Studio Code (VSCode)：精美的图形文本编辑器。它有一些非常有用的扩展，包括远程SSH扩展，允许您在VSCode本身中通过SSH编辑文件，而不是使用基于终端的编辑器。但是，过去一些学生遇到过复杂的设置问题。

在来实验室1之前，请确保您熟练使用所选文本编辑器在hive上编辑文件。

获取您的实验室仓库

访问Gradar，启动名为Labs的作业，并创建一个仓库。这将是您在整个学期中进行任何实验室工作的个人仓库。

配置Git(Configuring Git)

在我们开始之前，让我们告诉Git您是谁。这些信息将用于签名和记录您的提交。如果您以前设置过Git，您可能不需要在本地机器上执行此操作，但您需要在您的教学账户上执行此操作。

在您的本地机器上，告诉Git您的姓名和电子邮件：

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com

SSH连接到hive机器，并在您的教学账户上运行相同的命令。

克隆您的仓库

Git有"本地"和"远程"仓库的概念。本地仓库位于您的终端会话所在的位置；如果您在SSH会话中，本地仓库是hive机器上的一个文件夹；如果您的终端会话在本地机器上，本地仓库位于本地机器的文件系统上。远程仓库（例如GitHub仓库）通常托管在互联网上。

您在GitHub上有一个实验室仓库，但本地没有（如果网站能自动访问您的本地文件，那会有点令人担忧）。要获取此仓库的本地副本，您可以使用git clone，它将根据远程仓库的信息创建本地仓库。

SSH连接到hive机器。在hive机器上，将仓库克隆到名为labs的文件夹中：

git clone git@github.com:61c-student/sp26-lab-USERNAME.git labs

请记住将sp26-lab-USERNAME替换为您的实际仓库名称！

探索您的仓库：Git

进入这个新文件夹（cd）。列出所有隐藏文件（ls -a）。您看到一个隐藏的文件/文件夹了吗？

确实有一个名为.git的文件夹。它的存在表明当前文件夹（包含.git的文件夹）持有Git仓库。

查看您仓库当前的远程仓库和状态：

git remote -v
git status

git clone已经发挥了一些神奇的作用——有一个名为origin的远程仓库，其URL指向您在GitHub上的实验室仓库！您在一个名为main的本地分支上，它"跟踪"origin/main（origin远程仓库上的main分支）。

注意：GitHub现在使用main作为默认分支名称，而不是master

在整个学期中，课程工作人员可能会更新起始代码以修复错误或发布新实验。要接收这些更新，您需要添加另一个远程仓库。

git remote add starter https://github.com/61c-teach/sp26-lab-starter.git

注意：我们的起始仓库是公开的，您没有写访问权限，因此在这种情况下您实际上应该使用未经身份验证的HTTPS克隆URL！

如果您想拉取更新后的起始代码，您需要执行以下命令：

git pull starter main

现在就试试吧！由于您刚刚开始实验，可能还没有更新可以拉取。

探索您的仓库：文件(Exploring Your Repo: Files)

您的labs仓库结构如下：

labs/ (current directory)
    lab00/

~/labs/lab00 $ ls

在路径中，.本身指的是当前目录。以下命令将列出lab00文件夹中存在的文件：

~/labs/lab00 $ ls
~/labs/lab00 $ ls .

~/labs/lab00 $ ls ..
lab00   tools
~/labs/lab00 $ ls ../..
<whatever's in your home folder, probably many files>
~/labs/lab00 $ cd ..
~/labs $ ls
lab00   tools

The following sequences of commands will all list the files in lab00:

~/labs $ cd lab00
~/labs/lab00 $ ls

~/labs $ ls lab00

~/labs $ cd tools
~/labs/tools $ ls ../lab00

~/labs $ cd lab00
~/labs/lab00 $ ls ../lab00/../lab00/../lab00

Fizzing and Buzzing

如果您开始做其他实验室，请确保在开始此练习之前将您的工作复制到其他地方，因为在此练习中您的工作可能会被覆盖。

  1. cd到您仓库中的lab00文件夹，并查看存在的文件（ls）。然后，运行以下命令初始化实验室：

    bash init.sh

    确保没有打印错误。

  2. 使用Vim打开code.py并查看fizzbuzz(num)函数。它应该：

    • 如果num是3的倍数，打印"num: fizz"
    • 如果num是5的倍数，打印"num: buzz"
    • 如果num是3和5的倍数，打印"num: fizzbuzz"
    • 如果num不是3或5的倍数，什么都不打印

    但是，如果您运行程序（python3 code.py），似乎没有发生这种情况！请仅通过编辑if语句来修复此错误。修复代码后，保存并使用git add和git commit添加和提交您的工作。

    在许多环境中，git commit会打开Vim来编辑提交消息。如果您对如何使用它感到困惑，请参阅附录中的Vim基础部分。

  3. 提交修复后，push您的工作。

    或者至少，尝试推送您的工作。您应该会遇到一个错误：

     ! [rejected]        main -> main (non-fast-forward)
error: failed to push some refs to 'github.com:61c-student/sp26-lab-username.git'
提示(hint): 更新被拒绝，因为当前分支的尖端落后于远程分支
提示(hint): 其远程对应分支。请先合并远程更改（例如
提示(hint): 'git pull ...')，然后再推送
提示(hint): 有关详细信息，请参阅 'git push --help' 中的 'Note about fast-forwards'（快进说明）

    如果您没有遇到错误，请从步骤1重新开始，或者如果持续发生这种情况，请联系课程工作人员。

    在整个学期中，您可能会遇到许多奇怪的错误。将它们分解成更小的部分会有所帮助，看看您能否找出每个部分在说什么。例如，让我们来分解这个错误：

    • "failed to push some refs to REPO_URL": 推送(push)失败
    • "the tip of your current branch is behind its remote counterpart": 远程仓库(repo)（在GitHub上）有您本地仓库(repo)没有的提交(commit)
    • "Integrate the remote changes (e.g. 'git pull ...') before pushing again": 我们需要告诉Git如何整合这些神秘的提交(commit)

    尝试使用git pull拉取远程更改。

    • 如果您收到"fatal: Need to specify how to reconcile divergent branches."，请运行git config --global pull.rebase false，然后再试一次拉取
    • 如果您收到"fatal: Not possible to fast-forward, aborting"，请尝试git pull --ff
    • 如果您收到"fatal: Need to specify how to reconcile diverent branches"，请尝试git pull --ff

    如果一切顺利，您应该会遇到另一个错误！这是预期的行为。

    Auto-merging lab00/code.py
CONFLICT (content): Merge conflict in lab00/code.py
Automatic merge failed; fix conflicts and then commit the result.

    哎呀，合并冲突：

    • "Merge conflict in lab00/code.py"：远程仓库和本地仓库都对lab00/code.py进行了提交
    • "Automatic merge failed"：Git尝试弄清楚如何整合提交，但无法做到
    • "fix conflicts and then commit the result"：看起来我们需要手动解决合并冲突！

    您可以检查git status：

    在分支(branch) main 上
您的分支(branch)和 'origin/main' 已经分叉(diverged)
并且分别有 1 个和 1 个不同的提交(commit)
  （使用 "git pull" 将远程分支(branch)合并到您的分支中）

您有未合并的路径(path)。
  （修复冲突并运行 "git commit"）
  （使用 "git merge --abort" 中止合并）

未合并的路径(path)：
  （使用 "git add ..." 标记解决）
  双方都修改了(both modified): code.py

处理文件

touch将创建一个包含您提供的文件名的空白文件。

touch example.txt

This will create a file named example.txt with nothing inside.

如果您想一步创建文件并添加文本，可以使用：

echo 'Your contents here' > example.txt

这将在当前目录中创建一个名为example.txt的文件。如果文件已存在，将被覆盖。该文件将包含Your contents here，不带引号。>符号接受一个参数，用于重定向打印到stdout的数据发送位置。在这里，我们将echo的输出重定向到名为example.txt的文件。

您可以使用cat或less命令查看文件内容。

cat example.txt
less example.txt

cat将example.txt的内容打印到您的终端。less打开一个基本查看器，允许您滚动和搜索。

scp - "Secure Copy"

The scp program is used for copying files between computers using the SSH protocol.

有时，您可能希望将单个文件或整个文件夹从hive机器获取到本地系统，反之亦然。您可以使用scp来做到这一点：

scp <source> <destination>

要指定远程源或目标，请使用username@host:path。要指定本地目标，只需使用path。例如：

scp hive1:~/some-folder/example.txt ~/Downloads/

Assuming my username is cs61c-???, the above command would connect to hive1 and copy ~/some-folder/example.txt on my instructional account to ~/Downloads/example.txt on my local machine.

If I wanted to copy the other direction (from my local machine to a hive machine) I would use:

scp ~/Downloads/example.txt hive1:~/some-folder/

scp默认只对文件有效。要复制文件夹，您需要告诉scp"递归"复制文件夹及其所有内容，可以使用-r标志：

scp -r hive1:~/some-folder ~/Downloads/

注意斜杠：写some-folder将复制文件夹本身及其内部文件，而some-folder/将只复制内部文件。

警告：在hive机器上运行scp（例如当您在SSH会话中时）通常不是预期行为。在hive机器上运行scp example.txt hive4:~/example.txt会将example.txt复制到...同一位置。您可能希望在本地终端会话中运行它！

tmux - "终端复用器(Terminal Multiplexer)"

tmux是一个终端程序，允许您在单个终端窗口中运行多个终端实例。您可以生成多个终端窗口并平铺它们（左右或上下分割）。

当tmux与ssh结合使用时，即使您关闭笔记本电脑或SSH会话意外断开，您也可以保持工作状态。

Let's try that now!

    SSH进入任何hive机器。

  1. 运行tmux new - 这将创建一个新的tmux会话。
  2. In this terminal window, try opening htop
  3. 关闭计算机上的终端窗口/应用程序——这模拟您关闭笔记本电脑或SSH会话突然断开。
  4. 在笔记本电脑上打开另一个终端窗口，重新SSH进入hive。
  5. 输入tmux attach -t 0 - 您应该会看到htop再次弹出，而无需重新启动htop——Tmux使您之前的终端会话保持活跃！
  6. 要退出tmux，请按ctrl + b，然后按x，然后按y。
  • 要"分离"并将tmux隐藏到后台，但仍保持该终端会话活跃，请按ctrl + b，然后按d。

Tmux有一个键（称为"前缀键"），允许您输入命令与Tmux交互。默认情况下，这是ctrl + b，但命令行工具的好处是您可以更改所有这些设置！——这里是一位工作人员用来起步的良好配置。但您应该自定义这些工具以最适合您的工作！所以不要犹豫，可以在网上搜索技巧、现有配置和窍门。
